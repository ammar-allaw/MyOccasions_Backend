# شرح الفكرة الأساسية لمشروع MyOccasions

## مقدمة
مشروع MyOccasions هو تطبيق Backend مبني على Laravel لتوفير نظام إدارة شامل لمقدمي الخدمات، القاعات، والجهات ذات الصلة في سياق المناسبات والحفلات. هدف المشروع هو بناء واجهة برمجية (API) موثوقة تسمح لتطبيقات الجوال أو الويب بالتعامل مع المستخدمين، أصحاب الحسابات، الأدوار، الأذونات، وإدارة الخدمات.

## الفئات الرئيسية في النظام

- **العملاء (Client)**
  - لديهم الوصول إلى خدمات مقدمي الخدمة.
  - يمكنهم البحث عن الخدمات والقاعات حسب الدور والمحافظة والأسعار والسعة.

- **مقدمو الخدمة (Service Providers)**
  - يمكنهم إضافة وتحميل معلوماتهم وصورهم.
  - يمكنهم إضافة خدمات وقاعات وغرف، وتحديثها أو حذفها.

- **المالك (Owner)**
  - يمتلك سلطة أعلى على النظام.
  - يمكنه إنشاء مقدمي الخدمات وإدارة الأدوار والأذونات.
  - يمكنه إدارة مستويات الوصول والتحكم بصلاحيات المستخدمين.

- **الأدوار والأذونات**
  - نظام مرن يسمح بتخصيص صلاحيات لكل دور.
  - يدعم CRUD للأدوار والأذونات.
  - يمكن ربط الأدوار بالمستخدمين والأذونات بالمستخدمين.

- **المحافظات والمناطق**
  - يدعم حفظ قوائم المحافظات والمناطق لكل محافظة.
  - يسمح بترشيح مقدم الخدمة أو القاعة حسب الموقع الجغرافي.

## الميزات الرئيسية

### المصادقة والتسجيل
- تسجيل مستخدم جديد.
- تسجيل الدخول والخروج.
- تغيير كلمة المرور للمستخدمين.
- إعادة تعيين كلمة المرور للمستخدمين المصرح لهم.

### إدارة مقدمي الخدمة
- الحصول على تفاصيل مقدم الخدمة.
- تحديث بيانات مقدم الخدمة.
- رفع صورة لمقدم الخدمة.

### إدارة الخدمات والقاعات
- إضافة خدمة جديدة لمقدم الخدمة.
- إضافة غرفة أو قاعة.
- تحديث بيانات الخدمة.
- حذف خدمة أو غرفة.
- الحصول على تفاصيل قاعة معينة.

### البحث والتصفية
- الحصول على القوائم حسب الدور.
- تصفية مقدمي الخدمة حسب الدور، المحافظة، التصنيفات، السعر، السعة.
- دعم البحث عن الخدمات عبر `main_key_id` وعرض النتائج بترقيم صفحات.

### إدارة الأدوار والأذونات
- إنشاء، قراءة، تحديث، حذف الأدوار.
- إضافة وحذف الأذونات المرتبطة بالأدوار.
- إدارة أدوار المستخدمين.
- إدارة أذونات المستخدمين المباشرة.

## مكونات المشروع

- `app/Http/Controllers/Api/`:
  - تحتوي على المتحكمات الرئيسية مثل `AppController`, `AuthController`, `HallController`, `OwnerController`, `ServiceController`, `ServiceProviderController`, `GovernmentController`.

- `app/Services/`:
  - تحتوي على الخدمات المنطقية للتعامل مع المصادقة، المستخدمين، مقدمي الخدمات، الخدمات، وغيرها.

- `app/Models/`:
  - تحتوي على النماذج الأساسية مثل `Client`, `ServiceProvider`, `Role`, `Permission`, `Government`, `Region`, `Room`, `Service`.

- `routes/api.php`:
  - نقطة انطلاق لمسارات API.
  - تستورد وتعتمد ملفات المسارات الجزئية في `routes/Api/`.

- `routes/Api/`:
  - تحتوي على المسارات الخاصة بالتوثيق، المالك، الخدمات، القاعات، مقدمي الخدمات، الحكومة والتطبيق العام.

## نمط عمل API

- يعتمد المشروع على `middleware` مثل `auth:api`, `auth:owner`, `auth.provider.or.owner`, و`can.access` لضمان أمان الوصول.
- تستخدم بعض النقاط النهائية موارد تحويل JSON (`Resources`) لتهيئة استجابة مرتبة.
- يدعم الترقيم (`pagination`) في نتائج البحث لتسهيل الاستعراض عبر الواجهات الأمامية.

## سيناريو الاستخدام العام

1. المستخدم يسجل أو يسجل دخول.
2. العميل يطلب قائمة الأدوار المتاحة أو مقدمي الخدمات حسب الدور.
3. يمكن للعميل تصفية النتائج حسب المحافظة، السعر، السعة، أو التصنيف.
4. يقدم مالك الخدمة صلاحيات ويدير أدوار المستخدمين والأذونات.
5. بإمكان مقدمي الخدمة إضافة خدمات وقاعات وتحديث بياناتهم.

## خاتمة
مشروع MyOccasions هو نظام API متكامل لإدارة مناسبات ومقدمي خدماتها في بيئة Laravel. يدعم آليات المصادقة المتقدمة، إدارة الأدوار والأذونات، إدارة الخدمات والقاعات، والبحث المبني على عوامل تصفية متعددة.
