# GitHub Copilot — Project Coding Instructions

These instructions apply to **all** code generated or suggested in this Laravel project.
Follow them strictly for every feature, fix, or refactor.

---

## 1. General Principles

- Write **clean, readable, self-documenting** code.
- Every class must have **one responsibility** (Single Responsibility Principle).
- Depend on **abstractions (interfaces), not concrete implementations** (Dependency Inversion).
- Keep controllers **thin** — no business logic inside controllers.
- No raw SQL; use Eloquent with proper relationships and scopes.
- Never repeat yourself — extract reusable logic into services or traits.

---

## 2. SOLID Principles (Mandatory)

| Principle | Application |
|---|---|
| **S** — Single Responsibility | Each class does one thing only |
| **O** — Open/Closed | Extend via interfaces, not by modifying existing classes |
| **L** — Liskov Substitution | Implementations must be interchangeable with their interfaces |
| **I** — Interface Segregation | Keep interfaces small and focused |
| **D** — Dependency Inversion | Inject interfaces via constructor, bind in `AppServiceProvider` |

## 3. Folder & File Structure

For every new resource (e.g., `Product`), create the following structure:

```
app/
├── Http/
│   ├── Controllers/Api/
│   │   └── ProductController.php          ← thin, delegates to service
│   ├── Requests/Product/
│   │   ├── StoreProductRequest.php
│   │   └── UpdateProductRequest.php
│   └── Resources/Product/
│       └── ProductResource.php
│
├── Models/
│   └── Product.php                        ← with scopes & relationships
│
├── Repositories/Product/
│   ├── ProductRepositoryInterface.php     ← contract
│   └── ProductRepository.php             ← Eloquent implementation
│
└── Services/Product/
    ├── ProductServiceInterface.php        ← contract
    └── ProductService.php                ← business logic, uses repository
```

## 4. Repository Pattern

### Interface — `Repositories/Product/ProductRepositoryInterface.php`
```php
<?php

namespace App\Repositories\Product;

interface ProductRepositoryInterface
{
    public function all(array $filters = []);
    public function findById(int $id);
    public function create(array $data);
    public function update($model, array $data);
    public function delete($model): bool;
}
```

### Implementation — `Repositories/Product/ProductRepository.php`
```php
<?php

namespace App\Repositories\Product;

use App\Models\Product;

class ProductRepository implements ProductRepositoryInterface
{
    public function __construct(private Product $model) {}

    public function all(array $filters = [])
    {
        return $this->model->filter($filters)->latest()->get();
    }

    public function findById(int $id)
    {
        return $this->model->findOrFail($id);
    }

    public function create(array $data)
    {
        return $this->model->create($data);
    }

    public function update($model, array $data)
    {
        $model->update($data);
        return $model->refresh();
    }

    public function delete($model): bool
    {
        return $model->delete();
    }
}
```

## 5. Service Pattern

### Interface — `Services/Product/ProductServiceInterface.php`
```php
<?php

namespace App\Services\Product;

interface ProductServiceInterface
{
    public function list(array $filters = []);
    public function show(int $id);
    public function store(array $data);
    public function update(int $id, array $data);
    public function destroy(int $id);
}
```

### Implementation — `Services/Product/ProductService.php`
```php
<?php

namespace App\Services\Product;

use App\Repositories\Product\ProductRepositoryInterface;

class ProductService implements ProductServiceInterface
{
    public function __construct(
        private ProductRepositoryInterface $repository
    ) {}

    public function list(array $filters = [])
    {
        return $this->repository->all($filters);
    }

    public function show(int $id)
    {
        return $this->repository->findById($id);
    }

    public function store(array $data)
    {
        return $this->repository->create($data);
    }

    public function update(int $id, array $data)
    {
        $model = $this->repository->findById($id);
        return $this->repository->update($model, $data);
    }

    public function destroy(int $id)
    {
        $model = $this->repository->findById($id);
        return $this->repository->delete($model);
    }
}
```

## 6. Controller — Thin & Clean

```php
<?php

namespace App\Http\Controllers\Api;

use App\Exceptions\Handler;
use App\Http\Controllers\Controller;
use App\Http\Requests\Product\StoreProductRequest;
use App\Http\Requests\Product\UpdateProductRequest;
use App\Http\Resources\Product\ProductResource;
use App\Services\Product\ProductServiceInterface;
use Exception;

class ProductController extends Controller
{
    public function __construct(
        private Handler $handler,
        private ProductServiceInterface $service
    ) {}

    public function index()
    {
        try {
            $products = $this->service->list();
            return $this->handler->successResponse(
                ['products' => ProductResource::collection($products)],
                true,
                'Products retrieved successfully',
                200
            );
        } catch (Exception $e) {
            return $this->handler->errorResponse(false, $e->getMessage(), null, 400);
        }
    }

    public function store(StoreProductRequest $request)
    {
        try {
            $product = $this->service->store($request->validated());
            return $this->handler->successResponse(
                ['product' => new ProductResource($product)],
                true,
                'Product created successfully',
                201
            );
        } catch (Exception $e) {
            return $this->handler->errorResponse(false, $e->getMessage(), null, 400);
        }
    }

    public function show(int $id)
    {
        try {
            $product = $this->service->show($id);
            return $this->handler->successResponse(
                ['product' => new ProductResource($product)],
                true,
                'Product retrieved successfully',
                200
            );
        } catch (Exception $e) {
            return $this->handler->errorResponse(false, $e->getMessage(), null, 400);
        }
    }

    public function update(UpdateProductRequest $request, int $id)
    {
        try {
            $product = $this->service->update($id, $request->validated());
            return $this->handler->successResponse(
                ['product' => new ProductResource($product)],
                true,
                'Product updated successfully',
                200
            );
        } catch (Exception $e) {
            return $this->handler->errorResponse(false, $e->getMessage(), null, 400);
        }
    }

    public function destroy(int $id)
    {
        try {
            $this->service->destroy($id);
            return $this->handler->successResponse(null, true, 'Product deleted successfully', 200);
        } catch (Exception $e) {
            return $this->handler->errorResponse(false, $e->getMessage(), null, 400);
        }
    }
}
```

## 7. Binding Interfaces in AppServiceProvider

Always bind every interface to its implementation inside `AppServiceProvider.php`:

```php
use App\Repositories\Product\ProductRepositoryInterface;
use App\Repositories\Product\ProductRepository;
use App\Services\Product\ProductServiceInterface;
use App\Services\Product\ProductService;

public function register(): void
{
    $this->app->bind(ProductRepositoryInterface::class, ProductRepository::class);
    $this->app->bind(ProductServiceInterface::class, ProductService::class);
}
```

## 8. Form Requests — Validation

- Always use dedicated `FormRequest` classes; **never** validate inside controllers.
- Place requests under `app/Http/Requests/{Resource}/`.
- Return Arabic-friendly messages when the project is Arabic-facing.

```php
<?php

namespace App\Http\Requests\Product;

use Illuminate\Foundation\Http\FormRequest;

class StoreProductRequest extends FormRequest
{
    public function authorize(): bool { return true; }

    public function rules(): array
    {
        return [
            'name'    => 'required|string|max:255',
            'name_en' => 'required|string|max:255',
            'price'   => 'required|numeric|min:0',
        ];
    }
}
```

## 9. API Resources

- Always transform model data through a `Resource` class; **never** return raw models.
- Place resources under `app/Http/Resources/{Resource}/`.

```php
<?php

namespace App\Http\Resources\Product;

use Illuminate\Http\Resources\Json\JsonResource;

class ProductResource extends JsonResource
{
    public function toArray($request): array
    {
        return [
            'id'          => $this->id,
            'name'        => $this->name,
            'name_en'     => $this->name_en,
            'price'       => $this->price,
            'status'      => $this->whenLoaded('orderStatusAble', fn() => $this->orderStatusAble?->status?->name_en),
            'created_at'  => $this->created_at?->toDateTimeString(),
        ];
    }
}
```

## 10. Eloquent Model Scopes

Use **local scopes** in models for reusable query logic. Never repeat `where()` chains.

```php
// In the Model
public function scopeActive(Builder $query): Builder
{
    return $query->whereHas('orderStatusAble', fn($q) =>
        $q->whereHas('status', fn($q2) => $q2->where('name_en', 'accepted'))
    );
}

public function scopeFilter(Builder $query, array $filters): Builder
{
    return $query
        ->when($filters['search'] ?? null, fn($q, $search) =>
            $q->where('name', 'like', "%{$search}%")
        )
        ->when($filters['status'] ?? null, fn($q, $status) =>
            $q->whereHas('orderStatusAble.status', fn($q2) =>
                $q2->where('name_en', $status)
            )
        );
}
```

## 11. Naming Conventions

| Item | Convention | Example |
|---|---|---|
| Controller | PascalCase + `Controller` | `ProductController` |
| Service Interface | PascalCase + `ServiceInterface` | `ProductServiceInterface` |
| Service Class | PascalCase + `Service` | `ProductService` |
| Repository Interface | PascalCase + `RepositoryInterface` | `ProductRepositoryInterface` |
| Repository Class | PascalCase + `Repository` | `ProductRepository` |
| Request | PascalCase action + `Request` | `StoreProductRequest` |
| Resource | PascalCase + `Resource` | `ProductResource` |
| Route names | snake_case, plural | `products.index` |

## 12. API Response Format

Always use the existing `Handler` response helpers for consistency:

```php
// Success
$this->handler->successResponse($data, true, 'message', $statusCode);

// Error
$this->handler->errorResponse(false, $message, null, $statusCode);
```

## 14. Rules Summary (Quick Reference)

- [ ] Controller only calls Service methods — no Eloquent inside controllers
- [ ] Service contains business logic — calls Repository
- [ ] Repository handles all DB queries — never in Services or Controllers
- [ ] All inputs validated in FormRequest classes
- [ ] All outputs transformed through Resource classes
- [ ] Interfaces bound in AppServiceProvider
- [ ] Reusable queries extracted as Model Scopes
- [ ] No hardcoded IDs — use named status lookups (e.g., `Status::where('name_en', 'accepted')->first()`)
- [ ] Wrap mutating operations in `DB::transaction()`
- [ ] Never expose raw exception messages in production responses
